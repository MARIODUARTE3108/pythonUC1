Algoritmo: a receita por trás de tudo

Um algoritmo é uma sequência finita e ordenada de passos para resolver um problema — pense numa receita de bolo:

  • A ordem importa: não asse antes de misturar.
  • Os passos não podem ser ambíguos: "bata 5 minutos", não "bata um pouco".
  • Tem que ter um fim garantido: a receita não pode rodar para sempre.

Guarde essa palavra: algoritmo. Vamos usá-la o curso inteiro.

O "E" exigente e o "OU" flexível

✈️ E (and) — embarcar no avião Passaporte E passagem E estar no horário. Falhou um item? Não embarca. O E é exigente: exige que tudo esteja certo.

Por que os loops existem — contador e acumulador

O problema: imagine imprimir "Bem-vindo" 100 vezes. Sem repetição, seriam 100 linhas de print. Com repetição (um loop, ou "laço de repetição"), são duas linhas — não importa se é 100 ou 100 mil vezes.

🏭 Analogia: a linha de produção Numa fábrica, o operário não reescreve o manual a cada peça — ele repete a mesma operação ("aparafusar") centenas de vezes. O loop é a linha de produção do código: a mesma instrução, executada muitas vezes, sem reescrever nada.
🚪 Analogia 2: contar quem entra na loja Você fica na porta e, a cada pessoa que entra, soma 1. Repetir "somar 1" é um loop; a variável que acumula a contagem é um contador.

🗣️ Que tarefas repetitivas você faz que um computador poderia fazer por você? (renomear 200 fotos, enviar o mesmo e-mail para 50 pessoas, somar uma coluna de planilha…). Tudo isso é loop — é onde o computador mais brilha, porque ele nunca cansa nem erra por tédio.

do...while — o que é, e por que Python não tem

O PTD do curso pede do...while. Vamos ser honestos e claros, como fizemos com o switch na Aula 5: o Python nunca teve do...while — pela mesma filosofia de simplicidade.

☕ Analogia: a máquina de café O while é: "se você quiser café, faça um". O do...while é: "faça um café e depois pergunte se quer outro". No segundo caso, você sempre toma pelo menos um — o bloco roda pelo menos uma vez, mesmo que a condição já comece falsa.

Como se escreve em outras linguagens (JavaScript / C / Java — só para reconhecer, não para decorar):

do {
    opcao = pedirOpcao();
} while (opcao != "sair");

O equivalente em Python: while True + break.

💼 Dicas profissionais Use for quando souber o número de repetições; while quando depender de uma condição. Todo while precisa de uma saída (contador que muda, ou break). 🐛 Curiosidade: de onde vem a palavra "bug"? ▾
Em 1947, a engenheira Grace Hopper encontrou uma mariposa (em inglês, "moth", mas o termo "bug" já existia informalmente) presa num relé do computador Harvard Mark II, e a colou no relatório com a nota "primeiro caso real de bug encontrado". Depurar um programa = "tirar o inseto".
⚙️ Curiosidade: o range() não guarda tudo na memória de uma vez ▾
Diferente do que parece, range(1000000) não cria um milhão de números na memória imediatamente — ele os gera sob demanda, um de cada vez. Isso economiza muita memória em loops gigantes.
🐍 Curiosidade: Python tem um for...else raro ▾
Poucas linguagens têm isso: o bloco else de um for roda se o loop terminar sem ter usado break. É pouco usado no dia a dia, mas vale saber que existe.
